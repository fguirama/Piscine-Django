From Request to Response

When a user enters a URL in their browser, an HTTP request is sent to the web server. This request is first handled by the framework’s routing system, which determines which view should process it.

The view is responsible for handling the request. For a static page, the view simply selects a template and does not need complex logic or database access.

The template engine then renders the HTML template, optionally inserting data. Once rendered, the server sends the final HTML page back to the browser as an HTTP response.

Finally, the browser receives the response and displays the web page to the user.
