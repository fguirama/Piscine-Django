Django Template Engine

Django’s default template engine is designed to separate presentation from logic. It allows developers to generate dynamic HTML using a simple and readable syntax, while keeping Python code out of templates.

Blocks

Blocks are used with template inheritance. A base template defines replaceable sections using {% verbatim %}{% block %}{% endverbatim %}, and child templates override these sections to insert their own content.

Loops: for ... in

The {% verbatim %}{% for %}{% endverbatim %} loop allows iteration over lists or querysets passed from the view. Each iteration can display values using template variables.

if Control Structures

The {% verbatim %}{% if %}{% endverbatim %} tag enables conditional rendering. Content is displayed only if a condition evaluates to true, making it useful for handling optional data or user-specific content.

Context Variables

Context variables are passed from the view to the template as a dictionary. They are displayed using double curly braces, for example {{ variable }}, allowing dynamic data to appear in the rendered HTML.
